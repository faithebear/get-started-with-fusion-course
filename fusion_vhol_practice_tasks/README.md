# Fusion Practice Tasks README

Welcome to the Fusion Practice Tasks.

These exercises are designed to build your debugging instincts and fluency with dbt Fusion.

## How to Approach These Tasks

* **Complete the tasks in order.**
  Each task builds on concepts introduced earlier.

* The problems are **intentionally structured to become less guided** as you progress:

  * **Task 1** is more explicit about what to fix and what to look for.
  * **Task 2** provides fewer hints and requires stronger grain awareness.
  * **Task 3** is the most open-ended and expects you to rely on Fusion’s feedback and your data modeling intuition.

* Use Fusion intentionally:

  * Read real-time error messages carefully.
  * Hover to confirm column names and function signatures.
  * Preview CTEs to validate grain and logic.
  * Don’t just fix errors — validate the business logic.

## Goal

By the end of these exercises, you should be comfortable:

* Debugging compile-time and SQL errors quickly.
* Identifying grain issues using CTE previews.
* Verifying business logic beyond “it runs.”
* Using Fusion’s live feedback loop to iterate efficiently.

Take your time. Fix thoughtfully. Preview often.

Good luck — and try not to skip ahead.
